### **Cell 3: Judgement Generation**

- Now you have a noisy judge that works as follows:
- you randomly pick 3 ransom datapoints (a,b,c) from the dataset for n=20 times
- not that, a,b,c are all pairs themselves or 2d coordinates
- the noisy judge says which pair (a,b), (b,c), (a,c) is the closes to each other. 
- This results in an outcome like  ((a,b),c) meaning that a,b is the closest pair among the three
- to actually come up with the data you compute the three distances a_b, a_c, b_c, and then compute a Bradley–Terry type scoring.
- That is, the probabilty of a_b being selected it the probabilty is proportional to exp(- a_b). 
- add an optional temperature parameter that is by default 1 (has no influence).  
- Finally plot the judgements. Therefore, first plot the dataset using a scatter plot, with the 3 clusters having 3 shades of gray.
- then for each judgement  ((a,b),c)  you add 3 edges. 2 dashed edges with low opacity beween a,c and b,c representing datapoints that are not likely to be in a cluster.
- and then a real edge betweeb a,b representing that these might be in a cluster.
- use a differnt color for each judgement ((a,b),c), resulting in n=20 differnt colors being used.
- save the judgements as a csv file.
